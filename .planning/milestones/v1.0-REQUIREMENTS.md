## v1 Requirements — All Complete ✓

### Performance

- [x] **PERF-01**: Memory usage stays bounded during ZIP assembly — files streamed one-at-a-time from IndexedDB instead of all loaded at once
  - *Outcome: Implemented in Phase 1 (01-02). JSZip async generation processes files sequentially from IndexedDB cursor.*
- [x] **PERF-02**: Pre-compressed image files (JPEG, PNG, GIF) are stored without re-compression in the EPUB ZIP
  - *Outcome: Implemented in Phase 1 (01-02). Per-file compression mode: STORE for image types, DEFLATE for text.*
- [x] **PERF-03**: IndexedDB cache check and file fetch are combined into a single cursor pass
  - *Outcome: Implemented in Phase 1 (01-01, 01-02). getCachedFiles() returns cached content directly; single IDB open.*

### Security

- [x] **SEC-01**: All `onMessage` handlers validate `sender.id` before processing messages
  - *Outcome: Implemented in Phase 3 (03-01). Guard block at line 24 of background.js rejects unknown sender IDs.*
- [x] **SEC-02**: HTML content is sanitized using `DOMParser` instead of regex substitution
  - *Outcome: Implemented in Phase 3 (03-02). cleanHTML() uses DOMParser.parseFromString(); script elements removed structurally.*
- [x] **SEC-03**: API-derived strings inserted into XML templates are escaped for `<`, `>`, `"`, `&`
  - *Outcome: Implemented in Phase 3 (03-02). escapeXml() helper applied to opfPath in generateContainerXml(). Also applied sanitizeZipPath() for ZIP path traversal (WR-01 from code review).*

### Bugs

- [x] **BUG-01**: Object URLs created in `saveFile` are always revoked, even on download failure
  - *Outcome: Implemented in Phase 2 (02-02). try/finally block ensures URL.revokeObjectURL() always fires.*
- [x] **BUG-02**: Stylesheet `<link>` tags are preserved in `cleanHTML` (only script tags removed)
  - *Outcome: Implemented in Phase 3 (03-02). querySelectorAll('script') targets only scripts; link elements untouched.*
- [x] **BUG-03**: JWT cookie value is URL-decoded before use in the Authorization header
  - *Outcome: Implemented in Phase 2 (02-01). decodeURIComponent() applied to raw cookie value in getJWTToken().*
- [x] **BUG-04**: Rebuild from Cache mode skips network fetches entirely when all files are cached
  - *Outcome: Implemented in Phase 2 (02-03, 02-05). rebuildOnly flag wired through; file manifest stored/retrieved from IDB.*

### Reliability

- [x] **REL-01**: `activeDownloads` state is persisted to `browser.storage.session` so worker restarts don't kill in-flight downloads
  - *Outcome: Implemented in Phase 1 (01-02, 01-03). Session write-through on every progress update; startup restore reads session storage.*
- [x] **REL-02**: `downloadEPUB()` receives a progress callback instead of reaching into background.js globals
  - *Outcome: Implemented in Phase 2 (02-03, 02-04). sendProgress passed as explicit closure parameter.*
- [x] **REL-03**: OURN type fallback in `getBookMetadata` uses an explicit two-attempt structure with no recursion
  - *Outcome: Implemented in Phase 2 (02-02). Non-recursive sequential fallback: try book OURN, then try isbn-based OURN.*
- [x] **REL-04**: Dead `GET_BOOK_INFO` handler in `background.js` is removed
  - *Outcome: Implemented in Phase 2 (02-04). Handler removed; no callers in popup.js.*
