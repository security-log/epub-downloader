```javascript
    // rebuildOnly: assemble EPUB from IndexedDB, no network (BUG-04, D-03, D-04)
    if (options.rebuildOnly) {
      sendProgress(10, 100, 'Checking cache...');

      const manifest = await BookCache.getFileManifest(bookOurn);
      if (!manifest) {
        throw new Error(
          'Rebuild failed: no file manifest found for this book. ' +
          'Download the book normally first to populate the cache.'
        );
      }

      const cachedPaths = await BookCache.getCachedFilePaths(bookOurn);
      const missingFiles = manifest.filter(url => !cachedPaths.has(url));
      if (missingFiles.length > 0) {
        throw new Error(
          `Rebuild failed: ${missingFiles.length} file(s) missing from cache:\n` +
          missingFiles.join('\n')
        );
      }

      sendProgress(50, 100, `Building EPUB from ${cachedPaths.size} cached files...`);
      // buildEPUB loads all cached files directly from IndexedDB
      const epubBlob = await buildEPUB(zip, bookOurn);
      sendProgress(95, 100, 'Saving file...');
      const filename = sanitizeFilename(`${metadata.title}-${metadata.isbn}.epub`);
      await saveFile(epubBlob, filename);
      await addToHistory(metadata, bookOurn, filename, []);
      sendProgress(100, 100, 'Download complete!');
      return { success: true, filename, failedFiles: [], fromCache: cachedPaths.size };
    }
```

After this block, the normal download flow continues unchanged (`getAllFiles` result was already stored above, so here we proceed with `filesToDownload` assignment, `downloadAllFiles`, and `buildEPUB` as before).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| download.js → background.js | onProgress callback crosses from download.js into background.js scope; callback is caller-supplied |
| IndexedDB manifest → rebuildOnly diff | Stored manifest URLs are compared against cachedPaths; both come from extension-scoped storage |
| IndexedDB cached files → buildEPUB (rebuildOnly) | Cached file contents read back from IndexedDB and written to ZIP without re-validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Tampering | onProgress callback | accept | Callback is supplied by background.js (trusted extension code), not from web content. Phase 3 SEC-01 validates message senders upstream. |
| T-02-03-02 | Information Disclosure | rebuildOnly reads IndexedDB | accept | IndexedDB is extension-scoped storage; no cross-origin read possible in this context. |
| T-02-03-03 | Denial of Service | rebuildOnly with empty/missing cache | mitigate | Manifest existence check and missing-file diff both throw before ZIP assembly; error messages are user-actionable. |
| T-02-03-04 | Information Disclosure | missing-file error lists file paths | accept | File paths are O'Reilly internal API paths already known to the authenticated user who downloaded the book. No credentials or PII are included. |
</threat_model>
